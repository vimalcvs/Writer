Getting Started

Android Studio Project

Before starting to use Android Studio source code and developing app you need to first install the following softwares to your computer. If you have already installed these softwares you can skip this step.

  1. Install Java JDK
  2. Install Android Studio
  3. Now you should be able to open/edit the Android project and build APK

App Configuration

This chapter describes how to configure the project to be ready for publishing. All these steps are very important!

1. Import

Unzip the package and import/open the project in Android Studio.

2. Change Package Name

Rename all occurances of the packagecom.alim.writer
to your desired package Name ie., com.example.writer

3. Change Application Name

Go to res/values/ directory and edit strings.xml file.
Enter the name in the below string.
<string name="app_name">Writer</string>

4. Create Launcher Icon

Replace ic_launcher.png file in mipmap-mdpi, mipmap-hdpi, mipmap-xhdpi, mipmap-xxhdpi directories. See Android Cheatsheet for Graphic Designers for correct launcher icon dimensions.

  • Replace splash.png

Another possibility is to create launcher icons using Android Asset Studio.

Important: ic_launcher.png is the launcher icon and app splash logo/image.

Implement Admob

This chapter describes how to replace admob ad id in your application. All these steps are very important!

1. Getting the ad ID

     A. Create a admob account
     B. Create a application
     C. Create a native and interstitial ad

2. Change Admob application ID

Go to manifests directory and edit AndroidManifest.xml file.
Enter the name in the below string.

<meta-data

     android:name="com.google.android.gms.ads.APPLICATION_ID"
     android:value="Your admob Application ID" />

 

3. Change App ID

Go to res/values/ directory and edit strings.xml file.
Enter the name in the below string.
<string name="admob_app_id">ca-app-pub-3940256099942544~3347511713</string>

 

4. You should not add your ad id's here. You can do it in your admin application

Implement Firebase

This chapter describes how to implement firebase database in your application. All these steps are very important!

1. Getting google-services.json File

 

     A. Create a firebase account at https://console.firebase.google.com/
     B. Then click on Ad Project
     C. Enter project name (Example : Writer)
     D. Choose or create google analytics account
     E. Click on Create project
     F. Finally Download the google-service.json file

2. Import google-services.json File

 

Go to project view and select Project from the drop down list
Go to writer/app and paste google-services.json there

By adding  google-services.json all the firebase services like analytics, performance, crashlytics, database & storages will get connected to your firebase project.

Google Authentication

 

This chapter describes how to replace admob ad id in your application. All these steps are very important!

 

If you haven't yet specified your app's SHA-1 fingerprint, do so from the Settings page of the Firebase console

 

1. Authenticate your client using google auth

  • Open a terminal and run the keytool utility provided with Java to get the SHA-1 fingerprint of the certificate. You should get both the release and debug certificate fingerprints

  • To get the release certificate fingerprint :



    keytool -list -v -keystore {keystore_name} -alias {alias_name}

    Example : 

    keytool -list -v -keystore C:\Users\MG\Desktop\test.jks -alias test

    On Windows :

    When keytool command is not found, Go to your installed JDK Directory e.g. <YourJDKPath>\Java\jdk1.8.0_231\bin\, open command line and try the above commands for debug/release mode.

     

1. Enable Google Sign-In in the Firebase console

  • Open In the Firebase console, open the Auth section.

  • On the Sign in method tab, enable the Google sign-in method and click Save.

2. You must pass your server's client ID to the requestIdToken method. To find the OAuth 2.0 client ID

 

  • Open the Credentials page in the GCP Console.
  • The Web application type client ID is your backend server's OAuth 2.0 client ID.
  • Copy the Client ID and paste it in clientID at  com.alim.writer\Config\ConstantConfig

 

And you are ready to build your apk-release.apk

Build & Publish

A unique signed application needs to be published. To do that:

  1. Left click on the project and go to Android Tools
  2. Click on the "Export Signed Application"
  3. Create a new keystore
  4. Set the password
  5. Set you alias
  6. Select the location of where to save the apk


That's it.
